# Cursor 优先级测试

测试 `.cursorrules` 文件中规则的优先级顺序。

## 测试方法

创建一个测试文件：

```markdown
# 规则 A（在前面）

使用 snake_case 命名变量

---

# 规则 B（在后面）

使用 camelCase 命名变量
```

然后询问 Cursor："我应该使用什么命名规范？"

## 预期结果

- **如果 Cursor 使用"后面优先"**：会回答 camelCase
- **如果 Cursor 使用"前面优先"**：会回答 snake_case
- **如果 Cursor 使用"合并所有"**：可能提到两种都行或优先提某一种

## 当前实现的假设

当前代码注释说 "高优先级规则放在前面"，这意味着我们假设：

- ✅ **前面的规则优先级更高**
- ✅ `sortOrder: "desc"` → `[high, medium, low]` → high 在前面

## 问题

**如果 Cursor 实际是"后面覆盖前面"**（类似 CSS 的层叠规则），那么：

- ❌ 当前配置有问题
- ✅ 应该改为 `sortOrder: "asc"` → `[low, medium, high]` → high 在后面

## 社区说法

FAQ 提到 "社区经验无官方文档"，这说明：

1. 没有官方文档确认 Cursor 的规则优先级
2. 需要实际测试验证

## 建议

在添加单元测试前，先确认：

1. Cursor 的实际行为（前面优先 vs 后面优先 vs 都考虑）
2. 如果是"后面优先"，需要调整所有预设适配器的默认排序配置
