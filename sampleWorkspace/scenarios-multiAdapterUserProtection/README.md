# Scenarios: Multi-Adapter + User Protection

测试多适配器配置和用户规则保护的工作区。

## 夹具说明

- `rules/` 目录中的文件是**已提交的基线夹具**，用于表示目录型适配器生成后的预期工作区状态。
- 这些文件不是“测试跑脏了后遗留的随机产物”，而是这个场景的一部分。
- 如果在该工作区做手工测试并改动了 `rules/` 或 `.vscode/settings.json`，下一轮测试前应恢复到仓库提交态，或使用测试快照/清理逻辑恢复。

## 测试场景

- 多个自定义适配器（目录模式）
- 用户规则文件保护
- 孤立文件清理
- enableUserRules 配置
