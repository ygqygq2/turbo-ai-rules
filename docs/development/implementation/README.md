## 阅读建议

### 新开发者

建议按以下顺序阅读：

1. 先阅读 `docs/development/03-design.md` 了解整体架构
2. 阅读具体功能的实施文档了解实现细节
3. 参考流程图理解数据流和事件流

### 维护者

- 修改现有功能前，先查看对应的实施文档
- 添加新功能后，创建对应的实施文档
- 确保实施文档与代码保持同步

## 文档模板

创建新的实施文档时，建议包含以下部分：

1. **概述** - 功能简介
2. **背景** - 为什么需要这个功能
3. **问题分析** - 解决了什么问题
4. **解决方案** - 技术方案和架构设计
5. **实现步骤** - 详细的代码实现说明
6. **数据流** - 数据和事件的流转过程
7. **测试覆盖** - 测试场景和方法
8. **已知限制** - 当前的限制和问题
9. **未来改进** - 可优化的方向
10. **参考** - 相关文档和资源链接

## 维护原则

1. **及时更新**：功能实现后立即创建或更新实施文档
2. **详细记录**：记录关键决策和权衡考虑
3. **代码同步**：确保文档与代码实现一致
4. **清晰明了**：使用流程图、序列图等可视化工具
5. **问题导向**：说明解决了什么问题，为什么这样解决
